# BA 案例 - 需求分析案例库

## 案例1: /opsx:explore — 模糊的业务需求

**用户输入**: "我们想做一个用户行为分析功能"

**BA ★ 主导 — 反问澄清**:

❌ 不要直接说"那我们用 ClickHouse + Grafana"
✅ 反问澄清：

在出方案前，我需要先理解清楚：

【业务目标】
1. 这个分析功能服务谁？运营、产品、还是用户自己？
2. 想回答什么问题？比如"用户为什么流失" vs "哪个按钮点击多"，方向完全不同
3. 现在没有这个功能，业务上具体卡在哪？

【范围边界】
4. 只看 Web 端还是包括 App、小程序？
5. 历史数据要追溯多久？

【时效性】
6. T+1 报表够用，还是要实时？

**SA 辅助 — 技术可行性初判**:

- 调研现有数据采集链路是否能复用
- 列出"如果做，技术上的几个大方向"（埋点 SDK 自研 vs 第三方、存储选型范围）
- 不下结论，只列选项和约束

**输出**: 《需求探索纪要》
- 澄清问题清单
- 已知约束
- 未知项
- 技术可行性初判

## 案例2: 旅游攻略需求分析

**PM 输出**: 关键字拆解 - 6月18日/三天两夜/九江/南昌/旅游/攻略

**需求列表**:
1. [P0] 九江→南昌交通方案（高铁/大巴/自驾）
2. [P0] 九江住宿1晚 + 南昌住宿1晚
3. [P0] 九江核心景点游览（庐山至少半天）
4. [P0] 南昌核心景点游览（滕王阁必去）
5. [P1] 当地美食推荐
6. [P1] 预算估算
7. [P1] 6月天气应对（梅雨季/防暑）
8. [P2] 购物/特产推荐
9. [P2] 夜生活推荐

**验收标准**:
- 行程时间可执行（不赶路）
- 交通接驳可行（时刻表真实）
- 住宿位置合理（靠近景点/交通）
