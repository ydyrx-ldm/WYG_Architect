# TE 信号 - 测试验证预警信号

## 测试失败信号

- 模拟执行中时间不够 → 行程安排过紧
- 功能点未实现 → DEV 遗漏
- 信息不准确 → 事实核查不足
- 异常场景崩溃 → 错误处理缺失

## 回退频率信号

- DEV 回退超过3次 → 实现质量系统性不足
- SA 回退超过2次 → 方案可能有根本性缺陷
- BA 回退 → 需求理解可能有问题
- PM 回退 → 任务拆解可能遗漏关键点

## 错误记忆信号

- 同类错误记忆快速增长 → 系统性质量问题
- 错误记忆跨度多个Agent → 问题根源可能在上游
- 错误记忆未改善回退频率 → 错误记忆未被有效利用

## Bug 修复类验收信号

- 弱网/异常场景数据丢失 → 缺少持久化兜底
- 修复后回归测试失败 → 修复引入新问题
- 产线真实数据与测试数据结果不一致 → 测试覆盖不足
- 监控指标 24h 内波动 → 修复可能不彻底

## 方案偏离信号

- 实际实现的技术栈与方案设计不一致 → 方案偏离
- 关键组件被替换但无 propose 阶段的审批记录 → 擅自变更
- 实现中使用了方案未提及的外部服务 → 范围蔓延
- 方案偏离 + CR 未发现 → CR 门禁失职
