# TE 模式 - 测试验证模式库

## 模式1: 模拟执行测试法

```
1. 按用户视角走完整个流程
2. 在每个步骤检查：
   - 输入是否正确
   - 输出是否符合预期
   - 时间/资源是否在约束内
3. 记录所有偏差
```

## 模式2: 边界值测试

| 边界类型 | 测试方法 | 示例 |
|---------|---------|------|
| 空值 | 输入为空 | 空日志文件 |
| 极大值 | 输入超过上限 | 2GB日志文件 |
| 极小值 | 输入刚过下限 | 1行日志 |
| 并发 | 多用户同时操作 | 同时分析多个Issue |
| 异常 | 外部依赖不可用 | Jira API超时 |

## 模式3: 回退目标判定

```
问题是否影响具体实现？
  ├── 是 → 回退到 DEV
  └── 否 → 问题是否影响方案设计？
              ├── 是 → 回退到 SA
              └── 否 → 问题是否影响需求定义？
                          ├── 是 → 回退到 BA
                          └── 否 → 回退到 PM
```
