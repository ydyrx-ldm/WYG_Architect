# RR 模式 - 就绪评审模式库

## 模式1: 硬约束检查清单

| 约束 | 检查要点 | 判定标准 |
|------|---------|---------|
| 执行快速 | 方案是否简洁高效 | 无冗余步骤，关键路径清晰 |
| 用户友好 | 方案是否考虑用户体验 | 有错误提示、有备选方案、交互简洁 |

## 模式2: 准入标准升级路径

```
错误记忆 → 反复出现 → 升级为占位约束 → 具化指标 → 升级为硬约束
```

示例：
1. 多次出现"方案未考虑并发"的错误记忆
2. 升级为占位约束："并发处理标准 TBD"
3. 具化指标："支持100并发，响应时间<200ms"
4. 升级为硬约束

## 模式3: 回退原因分类

| 回退原因 | 典型表现 | 频率 |
|---------|---------|------|
| 方案不完整 | 遗漏关键功能 | 高 |
| 不可行 | 技术或资源限制 | 中 |
| 不满足硬约束 | 性能/体验不达标 | 中 |
| 缺少备选方案 | 无容错能力 | 低 |
