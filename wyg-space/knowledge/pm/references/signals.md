# PM 信号 - 项目管理预警信号

## 需求信号

- 用户输入过于简短（少于10字）→ 需求可能不完整
- 用户使用模糊词汇（"大概"、"可能"、"差不多"）→ 需要澄清
- 用户需求包含矛盾（"快速"且"全面"）→ 需要权衡

## 进度信号

- 同一阶段回退超过3次 → 方案可能有根本性问题
- 流水线总耗时异常长 → 某个Agent可能卡住
- 错误记忆快速增长 → 系统性质量问题

## 风险信号

- 涉及外部依赖 → 不可控风险
- 时间约束与任务量不匹配 → 进度风险
- 技术方案无备选 → 单点风险
- 需求频繁变更 → 范围蔓延风险

## 同步信号

- 本地文件变更未同步 → 违反硬约束
- GitHub 推送失败 → 需要立即处理
- 合并冲突频繁 → 同步策略需要调整

## 归档信号

- 复盘要点泛泛而谈 → 教训不够具体，无法指导后续改进
- 教训未反馈到对应 Agent 知识库 → 同类问题会重复发生
- 可复用资产未明确归属 → 资产会流失
- 交付物清单不完整 → 归档不完整，后续无法回溯
- 用户说"开发完了就归档吧" → 可能跳过了归档前检查
- 未确认生产环境验证 → 带病归档风险
- 未确认监控告警配置 → 上线后问题无法及时发现
- 未确认运维文档更新 → 交接后无法维护
