# PM 信号 - 项目管理预警信号

## 需求信号

- 用户输入过于简短（少于10字）→ 需求可能不完整
- 用户使用模糊词汇（"大概"、"可能"、"差不多"）→ 需要澄清
- 用户需求包含矛盾（"快速"且"全面"）→ 需要权衡

## 进度信号

- 同一阶段回退超过3次 → 方案可能有根本性问题
- 流水线总耗时异常长 → 某个Agent可能卡住
- 错误记忆快速增长 → 系统性质量问题

## 风险信号

- 涉及外部依赖 → 不可控风险
- 时间约束与任务量不匹配 → 进度风险
- 技术方案无备选 → 单点风险
- 需求频繁变更 → 范围蔓延风险

## 同步信号

- 本地文件变更未同步 → 违反硬约束
- GitHub 推送失败 → 需要立即处理
- 合并冲突频繁 → 同步策略需要调整
