## 案例3: 旅游攻略项目

**用户输入**: "制作6月18日三天两夜的九江、南昌旅游攻略"

**关键字拆解**:
- "6月18日" → 时间约束：出发日期确定，需查天气预报
- "三天两夜" → 时长约束：3天行程，2晚住宿
- "九江" → 目的地1：鄱阳湖/庐山/浔阳楼/东林寺
- "南昌" → 目的地2：滕王阁/八一起义纪念馆/秋水广场
- "旅游" → 任务类型：观光+休闲，非商务
- "攻略" → 交付物：可执行的详细行程方案

**风险识别**:
- 6月梅雨季，天气不确定性高
- 三天两夜覆盖两个城市，时间紧凑
- 庐山游览需至少半天，与南昌行程可能冲突

## 案例2: AI日志分析系统

**用户输入**: "做一个AI日志分析系统"

**关键字拆解**:
- "AI" → 技术类型：需要LLM集成
- "日志" → 数据类型：非结构化文本数据
- "分析" → 任务类型：信息提取+模式识别+根因定位
- "系统" → 交付物：可运行的完整系统

**风险识别**:
- 日志格式多样化，解析复杂
- AI分析可能超时
- 大日志文件处理性能问题
