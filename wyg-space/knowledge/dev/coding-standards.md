# DEV 知识库 - 开发实现

## 核心方法论

### 编码/内容规范

#### 通用原则

- **简洁**：用最少的代码/文字表达完整含义
- **清晰**：命名/表述无歧义，意图明确
- **一致**：风格统一，符合项目既有规范
- **完整**：不遗漏方案中定义的任何功能点

#### 代码规范

- 遵循语言/框架的惯用写法
- 必要的注释，不写废话注释
- 错误处理完备，不吞异常
- 函数/模块单一职责

#### 内容规范

- 信息准确，来源可查
- 逻辑连贯，层次清晰
- 格式统一，易于阅读

### 模块组织原则

- 高内聚低耦合
- 单一职责
- 依赖方向明确
- 可独立测试

### 规格遵从

- 严格遵循 SA 的方案设计
- 严格满足 RR 的准入要求
- 如发现方案不可行，标记问题而非自行变更
