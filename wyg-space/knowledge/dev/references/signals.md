# DEV 信号 - 开发实现预警信号

## 实现质量信号

- 实现与方案不一致 → 可能偏离规格
- 遗漏方案中的功能点 → 不完整
- 错误处理缺失 → 健壮性不足
- 代码/内容重复 → 可能有抽象不足

## 规格遵从信号

- 自行修改方案中的关键决策 → 需要回退到 SA
- 遗漏 RR 硬约束的实现 → 不满足准入标准
- 实现了方案中未定义的功能 → 范围蔓延

## 回退修复信号

- 同一错误反复出现 → 错误记忆未被有效利用
- 修复引入新问题 → 修复不够谨慎
- 修复后仍不通过 → 可能需要回退到更早阶段

## 质量自保信号

- 未写单元测试就提交 CR → 质量责任推给下游
- 只覆盖正常路径未覆盖异常路径 → 测试不充分
- Bug 修复只缓解症状未修 root cause → 问题会复发
- PR 描述未写清 root cause → CR 无法判断修复正确性

## 方案偏离信号

- 擅自替换方案中的关键组件（如 ClickHouse → Elasticsearch）→ 架构级偏离，必须回 propose
- 实现中引入方案未定义的外部依赖 → 范围蔓延
- 发现方案不可行但继续硬实现 → 应停止并标记问题
- 用模糊描述掩盖偏离（"优化了一下"实际是换了技术栈）→ 需要细查
