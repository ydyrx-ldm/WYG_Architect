# SA 模式 - 架构设计模式库

## 模式1: 流水线架构

```
输入 → 阶段1 → 阶段2 → ... → 阶段N → 输出
```

适用场景：
- 任务有明确的阶段划分
- 每个阶段有独立的输入/输出
- 阶段之间有依赖关系

## 模式2: 主备方案模式

```
主方案 (推荐)
  ├── 备选方案A (主方案不可行时)
  └── 降级方案 (最坏情况)
```

适用场景：
- 存在外部不确定性
- 关键决策有风险
- 需要容错能力

## 模式3: 分层架构

```
展示层 → 业务层 → 数据层
```

适用场景：
- 系统有明确的关注点分离
- 各层可独立变化
- 需要可测试性

## 模式4: 事件驱动架构

```
事件源 → 事件总线 → 处理器 → 结果
```

适用场景：
- 异步处理需求
- 松耦合组件
- 实时响应需求
