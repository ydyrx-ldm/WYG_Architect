# CR 信号 - 代码评审预警信号

## 质量信号

- 必须修改项超过3个 → DEV 实现质量低
- 同类错误反复出现 → DEV 未利用错误记忆
- 大量规范性问题 → DEV 未遵循编码规范

## 安全信号

- 硬编码敏感信息 → 安全漏洞
- 未验证外部输入 → 注入风险
- 权限检查缺失 → 越权风险

## 完整性信号

- 方案功能点未全部实现 → 遗漏
- 异常场景未处理 → 健壮性不足
- 边界情况未考虑 → 潜在bug

## Bug 修复类评审信号

- 修复只缓解症状未修 root cause → 问题会复发
- 修复引入新风险 → 回归测试不足
- 测试未覆盖 root cause 场景 → 无法保证不再发生
- PR 描述未写清 root cause → 无法判断修复正确性

## 方案偏离信号

- 实现中的组件与方案设计不一致 → 可能擅自替换
- 引入方案未定义的外部依赖 → 范围蔓延
- DEV 用模糊描述掩盖偏离（"优化了一下"实际是换了技术栈）→ 需要细查
- 架构级变更（存储/框架/语言替换）→ 必须驳回回 SA
