# CR 模式 - 代码评审模式库

## 模式1: 评审维度矩阵

| 维度 | 检查项 | 严重度 |
|------|-------|--------|
| 准确性 | 事实错误、逻辑错误 | 必须修改 |
| 完整性 | 功能遗漏、边界缺失 | 必须修改 |
| 规范性 | 风格不一致、命名不清 | 建议修改 |
| 安全性 | 漏洞、信息泄露 | 必须修改 |
| 性能 | 明显性能问题 | 建议修改 |

## 模式2: 常见代码坏味道

| 坏味道 | 说明 | 修复建议 |
|--------|------|---------|
| 过长函数 | 函数超过50行 | 拆分为小函数 |
| 重复代码 | 相同逻辑出现多次 | 提取公共方法 |
| 魔法数字 | 代码中出现未解释的常量 | 提取为命名常量 |
| 过深嵌套 | if嵌套超过3层 | 提前返回/策略模式 |
| 吞异常 | catch块为空 | 至少记录日志 |

## 模式3: 评审意见模板

```
[分类] 问题描述
- 位置: 文件/行号/段落
- 现状: 当前实现
- 建议: 期望实现
- 理由: 为什么需要修改
```
